# 작업계획서: 이력서 분류 문서 작성

## 1. 개요

### 1.1 작업 목적
기존 이력서 문서들(통합이력서, 기술경력서, 이력서, 자기소개서)을 바탕으로 4가지 분류 체계에 따른 문서를 작성하여 경력 정보를 효과적으로 전달

### 1.2 현황
| 기존 문서 | 페이지 | 주요 내용 |
|----------|--------|----------|
| 통합이력서.pdf | 12p | 프로필, 기술역량, 경력, 프로젝트, 비전 종합 |
| 기술경력서.pdf | 19p | 기술 스택, 프로젝트 상세, 아키텍처 설계 |
| 이력서.pdf | 7p | 간략한 경력 및 프로젝트 요약 |
| 자기소개서.pdf | 7p | 지원동기, 도전사례, 책임, 비전 |

---

## 2. 제미나이 협의 결과

### 2.1 핵심 내러티브
> **"문제 해결사"** - 건설 도메인 문제를 IT/AI 기술로 직접 해결해온 전문가

### 2.2 분류 체계 결정
| 분류 | 형식 | 목적 |
|------|------|------|
| **1. 종합 역량 요약** | Executive Summary | 1분 내 핵심 파악 |
| **2. 성향** | 서술형 + 소제목 | 가치관/철학/리더십 증명 |
| **3. 건설관련 경력** | 표(Table) 중심 | 경력 한눈에 파악 |
| **4. IT&AI 경력** | 프로젝트 카드 형식 | Problem→Role→Tech→Outcome |

### 2.3 주요 제안 사항
- **IT/AI 통합**: IT 역량은 AI 구현의 '도구'이므로 분리 시 연결고리 약화
- **스토리 기반**: 단순 나열이 아닌 경험 기반 증명
- **정량적 성과**: "어떤 문제를 어떻게 해결했고 결과는 무엇인가" 중심

## 4. 각 문서별 상세 구성

### 4.1 종합 역량 요약 (Executive Summary)
**형식**: 표 + 짧은 서술

**구성 항목**:
- 한 줄 소개
- 핵심 정체성 (역할, 경력, 가치, 차별점)
- 핵심 역량 3가지
- 핵심 기술 스택
- 대표 성과
- 커리어 전환 스토리

### 4.2 성향
**형식**: 서술형 + 소제목

**구성 항목**:
1. 핵심 가치: "문제가 기술을 이끈다"
2. 문제 해결 접근 방식: 데이터 중심 설계
3. 학습 태도: 비IT 배경에서 독학, AI 활용 학습
4. 리더십 스타일: 위기 상황 대응, 책임감
5. 사용자 가치 중심 사고

**근거 사례**:
- VE 업무 자동화 도전 (2017)
- 협업 시스템 도입 주도 (2019-2022)
- 총괄 단장 대리 역할 (2023)
- 다중 에이전트 방법론 독자 구축 (2025)

### 4.3 건설관련 경력
**형식**: 표(Table) 중심

**표 구성**:
| 기간 | 프로젝트명 | 소속/역할 | 주요 성과 |
|------|----------|----------|----------|

**포함 경력**:
1. 세종시 복합커뮤니티센터 (2015-2016) - CM 실무자
2. 본사 기술팀 (2017-2019) - VE 전문가, 구조 기술지원
3. 삼일빌딩 리모델링 (2019-2020) - 공무 담당자
4. 본사 연구소 (2021) - 연구원, ISO9001
5. 명동N빌딩 리모델링 (2021-2022) - 공무 담당자
6. 대전 5개현장 통합CM (2023-2024.05) - 단장 보좌

**강조점**:
- 문제 발견 능력 (현장 비효율 포착)
- 리더십 (5개 현장 총괄 대리)
- IT 도입 주도 (Trello, REDMINE)

### 4.4 IT&AI 경력
**형식**: 프로젝트 카드

**카드 구성**:
```
## 프로젝트명 (기간)
- **Problem**: 해결하고자 한 문제
- **Role**: 담당 역할
- **Tech Stack**: 사용 기술
- **Outcome**: 성과 및 결과
```

**포함 프로젝트**:
1. VE 업무 자동화 시스템 (2017-2019) - Excel 기반
2. CM 업무관리 시스템 (2023-2024) - Obsidian + GPT + OCR
3. 희림 사이드 프로젝트 (2024) - RAG, 법령 DB
4. 제안서 AI 시스템 (2024-현재) - 다중 에이전트
5. WPM 프로젝트 관리 도구 (2025.11-현재) - TypeScript CLI

**강조점**:
- 프로덕트 완성 경험
- 최신 기술 적용 (RAG, 다중 에이전트)
- 독자적 방법론 구축

## 6. 핵심 원칙

1. **내러티브 일관성**: "문제 해결사" 스토리 유지
2. **근거 기반**: 단순 나열이 아닌 경험으로 증명
3. **정량적 표현**: 가능한 경우 수치로 성과 표현
4. **연결성**: 건설 경력 → IT/AI 전환의 자연스러운 흐름
