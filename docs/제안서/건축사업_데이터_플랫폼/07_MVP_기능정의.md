# 건축사업 데이터 플랫폼 MVP 기능 정의

## 1. MVP 목표

MVP의 목적은 모든 부서를 위한 완성형 플랫폼을 만드는 것이 아니다. 초기 목표는 "프로젝트 전환 구간에서 데이터 단절을 줄일 수 있는가"를 검증하는 것이다.

따라서 MVP는 다음 가설을 검증해야 한다.

- 하나의 전환 구간에서 문서, 이슈, 변경, 결정 이력을 연결할 수 있는가
- 인수인계 품질을 사람 의존 방식보다 개선할 수 있는가
- AI가 권한 범위 안에서 실무자의 탐색 시간을 줄일 수 있는가

## 2. 권장 진입 범위

가장 적합한 MVP 범위는 "전 생애주기"가 아니라 "하나의 인수인계 구간"이다.

권장 우선순위는 다음과 같다.

### 우선안 A. 설계사업 -> CM사업 인수인계

- 이 비전의 차별성이 가장 잘 드러난다.
- 동일 프로젝트가 사업 영역을 넘어 이동하는 문제를 직접 다룬다.
- 설계 문서, 검토 이력, 변경 배경이 CM 단계로 잘 이어지지 않는 문제를 해결할 수 있다.

### 우선안 B. 입찰 -> 실행 인수인계

- 대부분 조직에서 반복적으로 발생하는 공통 문제다.
- 운영 효율과 실무 체감 효과가 빠르게 나타날 수 있다.

## 3. 주요 사용자

MVP에서 다루는 주요 사용자는 다음과 같다.

- 부서 책임자: 인수인계 범위와 핵심 이슈를 관리
- 공무 또는 문서관리 담당자: 문서와 변경 이력 정리
- 프로젝트 실무자: 필요한 배경 정보와 관련 자료 조회
- 관리자: 권한 관리, 감사 로그, 추적성 관리

## 4. 핵심 사용자 시나리오

### 시나리오 1. 프로젝트 인수인계 준비

프로젝트 책임자는 다음 단계로 넘겨야 할 핵심 문서, 이슈, 변경, 회의, 승인 이력을 하나의 인수인계 단위로 묶는다.

### 시나리오 2. 연결 관계 확인

다음 단계 담당자는 특정 문서가 어떤 회의, 어떤 결정, 어떤 변경 요청과 연결되는지 확인한다.

### 시나리오 3. AI 브리핑

사용자는 "이 프로젝트에서 아직 해결되지 않은 핵심 변경 이슈는 무엇인가" 같은 질문을 하고, AI는 권한 범위 안에서 근거와 함께 요약 답변을 제공한다.

### 시나리오 4. 누락 점검

시스템은 인수인계 패키지에 필수 승인 기록이나 핵심 변경 관련 근거 문서가 누락되었는지 점검한다.

## 5. 필수 기능

### 5.1 프로젝트 기본 모듈

- 프로젝트 생성 및 기본 정보 관리
- 프로젝트 단계 정의
- 참여 조직 및 사용자 권한 설정
- 역할별 접근 제어

### 5.2 데이터 등록 및 관리

- 문서 등록 및 분류
- 문서 버전 관리
- 문서 유형 지정
- 단계 및 부서 태깅

### 5.3 연결 관계 관리

- 문서와 문서 간 관계 연결
- 문서와 이슈, 변경, 회의, 승인 간 관계 연결
- 중요 결정의 근거 데이터 연결
- 인수인계 대상 항목 묶음 관리

### 5.4 인수인계 모듈

- 인수인계 패키지 생성
- 인수인계 체크리스트
- 미결 이슈 목록
- 핵심 변경 이력 요약
- 승인 및 결정 이력 포함 여부 표시

### 5.5 AI 지원 기능

- 권한 기반 질의응답
- 인수인계 브리핑 자동 생성
- 누락 데이터 탐지
- 관련 자료 추천
- 응답 근거와 출처 표시

### 5.6 감사 및 추적 기능

- 누가 무엇을 등록, 수정, 조회했는지 기록
- 변경 이력 보존
- 출처와 버전 추적

## 6. MVP에서 제외할 기능

초기에는 아래 기능을 제외하는 것이 적절하다.

- 전사 모든 부서를 포괄하는 완전한 모듈 세트
- BIM 전면 통합
- 고급 원가/공정 시뮬레이션
- 예측 분석 기반 자동 의사결정
- 외부 협력사 전체를 포함한 복잡한 다자간 협업 구조
- 모든 기존 시스템의 완전 대체

## 7. 비기능 요구사항

MVP라도 다음 요구사항은 핵심이다.

- 역할 기반 권한 제어
- 문서 및 객체 버전 관리
- 감사 로그
- AI 응답의 출처 표시
- 원문 접근 제한 정책
- 검색 성능과 기본 사용성 확보

## 8. 성공 지표

MVP의 성공 여부는 기능 수보다 다음 기준으로 판단해야 한다.

- 인수인계 준비 시간 감소
- 인수인계 후 추가 질의 발생 횟수 감소
- 누락된 변경 또는 승인 정보 건수 감소
- AI 브리핑 활용 빈도 증가
- 실무자의 정보 탐색 시간 감소

## 9. MVP 산출물 정의

초기 파일럿에서 실제로 보여줘야 하는 결과물은 다음과 같다.

- 하나의 프로젝트 공간
- 문서, 이슈, 변경, 회의, 승인 이력이 연결된 데이터 구조
- 인수인계 패키지 화면
- 역할별 접근 권한
- 출처 기반 AI 질의응답

## 10. 핵심 판단

MVP의 핵심은 "많이 만드는 것"이 아니라 "가장 아픈 전환 구간을 제대로 연결하는 것"이다. 따라서 초기 제품은 하나의 구간을 깊게 해결하는 편이 맞다.

한 문장으로 정리하면 다음과 같다.

> MVP는 전사 통합 플랫폼이 아니라, 인수인계 데이터 연속성을 검증하는 최소 운영 제품이어야 한다.
