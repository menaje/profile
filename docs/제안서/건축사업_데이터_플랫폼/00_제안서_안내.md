# 건축사업 데이터 플랫폼 제안서 안내

## 1. 문서 묶음의 성격

이 폴더는 개인 메모용 기획 문서가 아니라, 다른 건축사사무소나 CM 조직에 제안할 수 있는 대외 제안서 패키지다. 따라서 문서 구성도 `비전 설명`, `제안 요약`, `도입 제안`, `기술 검토 자료` 순서로 나뉘어 있다.

## 2. 권장 전달 순서

처음 제안할 때는 아래 순서가 가장 자연스럽다.

1. `04_1페이지_제안요약.md`
2. `05_도입_제안서_초안.md`
3. `03_시스템_개념도.md`

기술 검토가 필요한 경우에는 다음 문서를 추가로 활용한다.

1. `07_MVP_기능정의.md`
2. `08_데이터모델_초안.md`

## 3. 문서별 역할

- `01_건축사업_데이터_플랫폼_제안개요.md`
  제안의 문제 정의와 전체 방향을 설명하는 상위 개요 문서
- `02_비전선언문.md`
  서비스 철학과 장기 방향을 강하게 전달하는 비전 문서
- `03_시스템_개념도.md`
  공통 데이터 체계, 역할 기반 모듈, AI 접근 구조를 시각적으로 설명하는 문서
- `04_1페이지_제안요약.md`
  의사결정자에게 짧게 보여줄 수 있는 요약본
- `05_도입_제안서_초안.md`
  고객사 제안용 중심 문서
- `06_대외설명_피치_요약.md`
  미팅이나 소개 자리에서 구두 설명용으로 쓰기 좋은 압축 문서
- `07_MVP_기능정의.md`
  초기 파일럿 범위와 제품 검증 기준을 정리한 문서
- `08_데이터모델_초안.md`
  공통 데이터 체계와 관계 구조를 기술 검토용으로 정리한 문서

## 4. 활용 메모

이 패키지는 "새 PMIS를 하나 더 만들자"는 제안으로 읽히지 않게 사용하는 것이 중요하다. 문서 설명의 중심은 다음 세 가지에 두는 것이 적절하다.

- 프로젝트 데이터 연속성
- 역할 기반 모듈 구조
- 보안 통제형 AI 정보 접근

## 5. 한 줄 정리

이 제안서 패키지는 건축사업의 부서별 업무 분절을, 프로젝트 중심 데이터 운영 구조로 바꾸기 위한 대외 제안 문서 묶음이다.
