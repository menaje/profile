# 건축사업 데이터 플랫폼 비전선언문

## 선언

우리는 건축사업이 부서별로 나뉘어 관리되더라도, 프로젝트의 데이터와 맥락은 끊기지 않아야 한다고 본다.

설계사업, CM사업, 입찰, 실행, 유지관리의 각 단계는 서로 다른 업무가 아니라 하나의 프로젝트가 시간에 따라 전개되는 과정이다. 그럼에도 오늘의 현장에서는 같은 프로젝트가 부서와 단계가 바뀔 때마다 다시 해석되고, 다시 정리되고, 다시 인수인계된다. 그 결과 데이터는 남아도 맥락은 사라지고, 문서는 전달되어도 의사결정의 배경은 누락된다.

우리가 만들고자 하는 것은 또 하나의 부서별 업무관리 도구가 아니다. 우리가 지향하는 것은 건축사업 전 생애주기를 하나의 데이터 체계로 연결하고, 각 부서가 자신의 역할에 맞는 방식으로 일하면서도 같은 프로젝트를 동일한 맥락 안에서 운영할 수 있게 만드는 플랫폼이다.

## 비전

> 건축사업의 전 생애주기를 하나의 데이터 맥락으로 연결하는 프로젝트 운영 플랫폼

## 미션

설계, CM, 입찰, 실행, 유지관리 전 과정의 데이터를 공통 체계 위에서 연속적으로 관리하고, 각 부서는 역할 기반 모듈을 통해 업무를 수행하며, AI는 권한과 보안을 전제로 필요한 정보만 해석하고 제공하도록 만든다.

## 우리가 만들고자 하는 변화

- 부서별 관리에서 프로젝트 중심 데이터 운영으로 전환한다.
- 문서 전달 중심 인수인계에서 맥락 전달 중심 인수인계로 전환한다.
- 단일 부서용 PMIS에서 프로젝트 전 생애주기 데이터 플랫폼으로 확장한다.
- 원문 탐색 중심 업무 방식에서 권한 기반 AI 정보 접근 방식으로 전환한다.

## 핵심 원칙

### 1. 데이터는 부서를 넘어 이어져야 한다

같은 프로젝트라면 어느 부서가 맡든 핵심 데이터 객체와 이력은 단절되지 않아야 한다.

### 2. 업무 방식은 달라도 데이터 원천은 연결되어야 한다

설계와 CM, 입찰과 실행, 유지관리는 서로 다른 화면과 프로세스를 가질 수 있다. 그러나 프로젝트를 구성하는 데이터의 원천과 관계는 하나의 체계 안에서 유지되어야 한다.

### 3. 인수인계는 문서 전달이 아니라 맥락 이전이어야 한다

무엇이 결정되었는지만이 아니라 왜 그렇게 되었는지, 누가 승인했는지, 어떤 변경을 거쳤는지까지 함께 이어져야 한다.

### 4. 보안은 활용의 반대가 아니라 설계의 출발점이어야 한다

모든 사용자가 모든 원문에 접근하는 구조는 현실적이지 않다. 따라서 권한, 역할, 책임 범위를 전제로 정보 접근 체계를 설계해야 한다.

### 5. AI는 대체자가 아니라 신뢰 가능한 접근 계층이어야 한다

AI는 원천 데이터를 대신하는 존재가 아니라, 권한 범위 안에서 필요한 정보와 관계를 찾아 설명하고 연결해 주는 인터페이스여야 한다. 모든 응답은 근거와 출처를 가져야 한다.

## 우리가 믿는 가치

- 프로젝트는 문서 묶음이 아니라 살아 있는 데이터 맥락이다.
- 좋은 시스템은 모든 사람에게 같은 화면을 강요하지 않는다.
- 복잡한 조직일수록 공통 데이터 체계가 필요하다.
- AI의 가치는 생성보다도 맥락 있는 접근과 해석에 있다.
- 장기적으로 경쟁력을 만드는 것은 기능 수가 아니라 데이터 연속성이다.

## 결론

이 비전은 기존 PMIS의 기능을 조금 더 추가하는 발상이 아니다. 그것은 건축사업을 바라보는 기준을 바꾸는 일이다.

우리는 각 부서가 따로 일해도 프로젝트는 하나의 흐름으로 이어져야 한다고 본다. 그리고 그 연결의 중심에는 화면이 아니라 데이터가, 데이터의 활용에는 검색이 아니라 맥락과 권한을 이해하는 AI가 있어야 한다고 본다.

이 플랫폼의 목표는 명확하다.

> 각 부서는 자신의 방식으로 일하되, 프로젝트는 하나의 데이터 맥락 안에서 운영되게 한다.
