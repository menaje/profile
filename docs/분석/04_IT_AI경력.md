## 핵심 전문성

### 다중 에이전트 오케스트레이션

여러 AI 에이전트를 조율하여 복잡한 작업을 수행하는 시스템 설계. 각 에이전트에게 명확한 역할을 부여하고, 협업 프로토콜을 통해 일관된 결과물 생성.

**핵심 원칙:**
1. 명확한 역할 분담: 각 에이전트의 책임과 권한 정의
2. 컨텍스트 공유: 일관성 유지를 위한 맥락 전달
3. 의존성 관리: Task 간 순서 및 병렬 실행
4. 피드백 루프: 결과 검증 및 재시도

---

### 컨텍스트 엔지니어링

LLM에 정확한 맥락을 제공하여 품질을 극대화하는 기법. 프롬프트 최적화, 단계별 작업 분할, 대용량 문서 생성 시 일관성 유지.

**핵심 기법:**
1. 단계별 분할 (Chunking): 대용량 작업을 Phase → Stage → Task로 분할
2. 컨텍스트 누적 (Context Accumulation): 이전 단계의 결과를 다음 단계로 선별적으로 전달
3. 프롬프트 최적화: 명확한 지시어, 예시 제공, 출력 형식 명시
4. 검증 및 재시도: 결과 검증 로직, 실패 시 피드백을 통한 재시도

---

### RAG (Retrieval-Augmented Generation) 시스템

Vector DB 기반 문서 검색 및 생성. 대규모 지식 베이스에서 관련 정보를 검색하여 LLM에 제공, 정확하고 최신 정보 기반의 답변 생성.
