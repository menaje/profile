# AI 업무 활용 방식에 대한 관점 정리

## 1. 문제의식

AI를 업무에 활용하는 방식은 모든 업무에서 동일하지 않다. 어떤 업무는 AI 자체의 생성 성능과 추론 성능이 좋아질수록 활용성이 급격히 높아지고, 어떤 업무는 모델의 성능보다도 업무 과정에서 생성되는 데이터를 얼마나 잘 수집하고 연결하고 제공하느냐가 더 중요하다.

이 관점은 업무에 AI를 적용할 때 무엇을 먼저 설계해야 하는지 구분하는 데 유효하다.

## 2. 관점 정리

### 2.1 AI 퍼포먼스가 중요한 영역

제안서 작성, 설계안 정리, 설계도면 보조 해석, 기획 초안 작성과 같이 결과물의 품질이 AI의 이해력, 생성력, 맥락 유지 능력에 크게 좌우되는 업무는 AI 성능의 향상에 따라 활용성이 직접적으로 높아진다.

이 영역에서는 다음 요소가 중요하다.

- 긴 맥락을 이해하고 유지하는 능력
- 복합 지시를 반영해 결과물을 재구성하는 능력
- 사람과 여러 차례 상호작용하며 초안을 개선하는 능력
- 텍스트, 표, 이미지, 도면 등 다양한 형식을 함께 다루는 능력

즉, 이런 업무는 좋은 데이터를 넣는 것만으로는 한계가 있고, 결국 더 나은 모델과 더 나은 협업 인터페이스가 실질적인 생산성 향상으로 이어진다.

### 2.2 데이터가 중요한 영역

반대로 CM과 같이 여러 업무 사이에서 발생하는 데이터를 기반으로 문서 행정과 관리 업무가 이루어지는 영역에서는, AI의 순수 생성 성능보다 데이터 관리와 활용 체계가 훨씬 중요하다.

이 경우 핵심은 AI가 스스로 훌륭한 문장을 만들어내는 것이 아니라, 이미 존재하는 데이터를 다음과 같이 다룰 수 있게 만드는 데 있다.

- 여러 부서와 단계에서 발생한 데이터를 수집한다.
- 문서와 기록을 구조화하고 분류한다.
- 필요한 시점에 근거와 함께 찾아서 제공한다.
- 서로 다른 업무 간 데이터를 연결해 맥락을 복원한다.

이런 업무에서는 AI가 "생성 주체"라기보다 "데이터 접근 인터페이스" 또는 "업무 맥락 복원기"로 작동할 때 가치가 크다.

## 3. 내 의견

나는 이 관점이 상당히 타당하다고 본다. 특히 AI 도입을 추상적으로 이야기하지 않고, "무엇이 가치의 원천인가"를 기준으로 나누려는 점이 좋다.

정리하면 다음과 같다.

- 제안서, 설계, 기획 초안 같은 업무는 AI의 퍼포먼스가 좋아질수록 활용 폭이 커진다.
- CM, 문서 행정, 이력 관리, 인수인계 지원 같은 업무는 데이터 체계가 정리될수록 활용성이 커진다.
- 실제 현장에서는 두 방식이 분리되지 않고 대부분 혼합된다.

다만 한 가지를 더 보완하면 좋겠다. 실제로는 `AI 성능`과 `데이터`만이 아니라 `업무 구조화와 거버넌스`가 세 번째 핵심 축이 된다.

왜냐하면 데이터가 많아도 다음 조건이 없으면 AI 활용성이 쉽게 떨어지기 때문이다.

- 문서와 데이터의 기준 키가 정의되어 있는가
- 버전과 변경 이력이 관리되는가
- 권한 체계가 분명한가
- 어떤 답변이 어떤 근거에서 나왔는지 추적 가능한가
- 업무 흐름 안에서 실제로 사용할 입력과 출력 지점이 설계되어 있는가

즉, 데이터 중심 업무에서 중요한 것은 단순히 "데이터를 많이 모으는 것"이 아니라, AI가 안전하게 읽고 연결하고 설명할 수 있도록 운영 구조를 만드는 일이다.

## 4. 실무적으로 보는 적용 방향

이 관점은 AI 도입 전략을 나눌 때 꽤 실용적이다.

### 4.1 퍼포먼스 중심 업무의 도입 방식

이 경우에는 다음이 우선이다.

- 더 나은 모델 선택
- 사람과 반복 협업할 수 있는 인터페이스
- 초안 생성 후 빠르게 수정하는 워크플로우
- 결과물 품질을 검토하는 사람 중심 검수 체계

핵심은 "AI가 얼마나 잘 생각하고 표현하느냐"에 있다.

### 4.2 데이터 중심 업무의 도입 방식

이 경우에는 다음이 우선이다.

- 데이터 수집 체계 정비
- 문서 분류와 메타데이터 표준화
- 프로젝트 단위의 공통 식별 체계 구축
- 검색, 조회, 요약, 근거 제시 흐름 설계
- 권한과 보안 정책 정비

핵심은 "AI가 무엇을 근거로 답하느냐"에 있다.

### 4.3 혼합형 업무의 도입 방식

실제 업무는 대부분 혼합형이다. 예를 들어 제안서 작성도 AI 성능이 중요하지만, 이전 프로젝트 이력, 조직 표준 문안, 기술 자료, 도면 정보가 잘 연결되어 있어야 결과 품질이 안정된다.

반대로 CM 문서 행정도 데이터가 핵심이지만, 상황 설명, 요약, 대외 문안 정리 단계에서는 모델의 성능이 중요해진다.

따라서 현실적인 방향은 다음과 같다.

- 생성이 중요한 구간에는 고성능 AI를 쓴다.
- 근거가 중요한 구간에는 데이터 체계와 검색 구조를 먼저 만든다.
- 최종적으로는 둘을 하나의 업무 흐름 안에서 연결한다.

## 5. 결론

당신의 관점은 AI 도입 논의를 꽤 본질적으로 나누는 기준이라고 본다. "AI가 똑똑해지면 해결되는 문제"와 "데이터가 정리되어야 해결되는 문제"를 구분하는 것은 매우 중요하다.

내 의견을 한 문장으로 요약하면 이렇다.

> 생성형 업무는 모델 성능이 지배하고, 운영형 업무는 데이터 구조가 지배하며, 실제 현장의 성패는 이 둘을 업무 거버넌스 안에서 어떻게 연결하느냐에 달려 있다.

따라서 앞으로 AI 활용 전략을 세울 때는 업무를 다음 세 가지로 분류해 보는 것이 좋다.

- 성능 지배형 업무
- 데이터 지배형 업무
- 혼합형 업무

이 구분이 선명해질수록 "어디에 모델 비용을 써야 하는지"와 "어디에 데이터 체계를 먼저 만들어야 하는지"가 같이 보이게 된다.
