## 1. 포지셔닝 전략

> "복잡한 현실 세계의 문제를 AI 시스템으로 해결하는 개발자"

| 구분 | 내용 |
|------|------|
| **경력 프레이밍** | 건설CM 9년 → "복잡계 문제 해결 경험 9년" |
| **총 경력** | 11년 (복잡계 문제 해결 9년 + AI 시스템 개발 2년) |
| **핵심 가치** | 실제로 동작하고 가치를 창출하는 AI 시스템 설계 |

## 2. 핵심 전문성

### 2.1 전문성 정의

| 기존 표현 | IT 시장 맞춤 표현 |
|----------|------------------|
| 데이터 중심 프로젝트 관리 | **LLM 에이전트를 위한 영속적 메모리 및 워크플로우 관리 시스템 설계** |
| AI 퍼포먼스 중심 다중 에이전트 오케스트레이션 | **복잡한 태스크 자동화를 위한 다중 에이전트 워크플로우 오케스트레이션** |

### 2.2 상위 개념

**"AI 네이티브 시스템 아키텍처 설계"**
- AI의 작동 방식을 근본적으로 이해
- AI가 최상의 성능을 내도록 전체 시스템(데이터, 에이전트, 워크플로우) 설계

## 3. 주요 프로젝트 (3개)

### 3.1 프로젝트 배치 전략

| 순위 | 프로젝트 | 비중 | 강조점 |
|------|----------|------|--------|
| 1 | **CONI** | 40% | AI 네이티브 시스템 설계 철학, 아키텍처 중심 |
| 2 | **다중 에이전트 오케스트레이션** | 35% | 비즈니스 성과 (20여 종 보고서 자동 생성) |
| 3 | **CM 프로젝트 관리 시스템** | 25% | 실제 현장 검증, 시스템 정착 경험 |

### 3.2 프로젝트 상세

#### CONI (40%)
- **유형**: 개인 프로젝트 → "핵심 역량 증명을 위한 프로토타입 프로젝트"
- **핵심**: 작업계획서 컨텍스트 + 메모리 기능을 활용한 맥락 중심 AI 협업
- **기술**: TypeScript, Node.js, Supabase, Vector DB
- **강조**: 아키텍처 설계 철학, AI 컨텍스트 유지 메커니즘

#### 다중 에이전트 오케스트레이션 시스템 (35%)
- **유형**: 회사 프로젝트 (희림종합건축사사무소)
- **핵심**: 한 번 실행으로 20여 종 보고서/슬라이드 자동 생성
- **기술**: Python, FastAPI, Gemini API, Supabase
- **강조**: 오케스트레이션 기술, 비즈니스 효율성

#### CM 프로젝트 관리 시스템 (25%)
- **유형**: 회사 프로젝트 (정림CM)
- **핵심**: 5개 현장 통합 문서관리, 현장 핵심 협업 시스템으로 정착
- **기술**: Obsidian, Python, GPT API, Gemini Vision
- **강조**: 도메인 문제 해결, 시스템 도입/확산 경험

## 5. 문서별 작성 계획

### 5.1 이력서 (A4 2-3p)

**목적**: 1분 내 핵심 역량 파악

**구성**:
1. 인적사항 + 한 줄 소개
   - "복잡한 현실 세계의 문제를 AI 시스템으로 해결하는 개발자"
2. 핵심 정체성 표
   - 현재 역할, 총 경력(11년), 핵심 가치
3. 핵심 전문성 2가지
   - LLM 에이전트를 위한 영속적 메모리/워크플로우 관리
   - 다중 에이전트 워크플로우 오케스트레이션
4. 기술 스택 요약표 (1등급 중심)
5. 주요 프로젝트 요약 (3개, 각 2-3줄)
6. 경력 요약표 (회사/기간/역할/핵심성과)
7. 학력/자격

**참고 자료**: `01_종합역량요약.md`, `03_건설관련경력.md`

### 5.2 기술경력서 (핵심 5p + 부록)

**목적**: 기술 역량과 프로젝트 경험 상세 증명

**[핵심 5p]**
1. 기술 스택 상세 (숙련도 표시)
2. 핵심 전문성 설명
   - AI 네이티브 시스템 아키텍처란?
   - 컨텍스트 엔지니어링 방법론
3. 주요 프로젝트 상세 (3개)
   - CONI (Problem-Role-Tech-Outcome, 아키텍처 다이어그램)
   - 다중 에이전트 오케스트레이션 시스템
   - CM 프로젝트 관리 시스템
4. 오픈소스 프로젝트
   - mlx-serve (Homebrew 배포, ML엔지니어링)
   - supamigrate (DB 마이그레이션 도구)

**[부록 3-5p]**
- 기타 프로젝트 (RAG 시스템, VE 자동화)
- 건설 경력 상세 (IT 도입 중심)
- 기술 성장 곡선 (Phase 1→2→3)

**참고 자료**: `04_IT_AI경력.md`, `03_건설관련경력.md`, README 파일들

### 5.3 자기소개서 (A4 1.5-2p)

**목적**: 가치관, 철학, 성장 스토리, 비전 전달

**제목**: "9년의 질문에 AI로 답을 찾아가는 아키텍트"

**구조**:
1. **도입** (역량 선언)
   > "저는 복잡한 현실의 문제를 해결하기 위해 AI 네이티브 시스템을 설계하고 구현하는 개발자입니다."

2. **문제의식** (경험 기반)
   - 9년간 현장에서 데이터와 컨텍스트 없이는 어떤 기술도 무용지물
   - "파편화된 데이터"와 "암묵지" 문제 발견

3. **핵심 도전 사례** (2개)
   - **CONI**: AI가 맥락을 유지하는 아키텍처 설계
     - 작업계획서 컨텍스트 + 메모리 기능
     - "대화가 끊겨도 작업은 이어진다"
   - **다중 에이전트 오케스트레이션**: 비즈니스 성과 창출
     - 한 번 실행으로 20여 종 보고서 자동 생성
     - 독자적 방법론 → 최신 기술과의 조우

4. **미래 비전**
   > "복잡하고 도전적인 IT 분야의 문제를 AI 시스템으로 해결하고 싶습니다."
   - 함께 만들고 싶은 미래: "데이터가 스스로 일하는 지능형 시스템"

**참고 자료**: `02_성향.md`, `01_종합역량요약.md`
