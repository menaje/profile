## Problem

문서 중심의 건설사업관리(CM) 업무는 본질적으로 정보가 분산됩니다. A 지자체 산하 약 5개 건설 현장을 동시에 관리하는 상황에서 다음과 같은 문제가 있었습니다.

- **정보 분산**: 필요한 자료를 찾고 맥락을 설명하는 데 과도한 시간이 소요되었습니다.
- **이기종 데이터 구조**: 현장마다 데이터 구조가 달라 협업과 의사결정이 지연되었습니다.
- **비IT 사용자 비율**: 현장 인력 대부분이 비IT 사용자로, 복잡한 시스템은 도입 자체가 실패할 가능성이 컸습니다.
- **반복 업무 부담**: 월간보고서 작성과 문서 등록 같은 반복 작업을 줄이지 않으면 시스템이 현장에 정착하기 어려웠습니다.

## Role

1인 기획·개발로 전체 시스템을 설계하고 현장에 안착시키는 역할을 담당했습니다.

- **문서관리 구조 설계**: Obsidian 기반 맞춤형 문서관리 구조를 설계하고 현장 업무 흐름에 맞게 적용했습니다.
- **AI 자동화 파이프라인**: OCR, LLM, 문서 등록 자동화를 결합해 비정형 문서(PDF, 이미지)를 구조화하는 파이프라인을 구축했습니다.
- **통합 대시보드**: 사업비, 계약, 문서 메타데이터를 연결해 현장별 상황을 빠르게 파악할 수 있는 실시간 가시성을 확보했습니다.
- **보고서 자동화**: 월간보고서 자동 생성 흐름을 구성해 실무자의 반복 작업을 대폭 줄였습니다.
- **현장 안착**: 도구 배포에서 끝내지 않고, 전체 협업 방식에 실제로 안착시키는 역할까지 담당했습니다.

## System

| 구분 | 기술 |
|------|------|
| **플랫폼** | Obsidian (Markdown 기반 지식 관리) |
| **언어** | Python |
| **AI/LLM** | OpenAI GPT API, Google Gemini API |
| **Vision/OCR** | Apple Vision → Gemini 멀티모달 모델 |
| **자동화** | Python 스크립트, Markdown 기반 구조화 |

### 기술적 도전과 해결

- **비정형 문서 구조화**: 건설 현장의 문서는 PDF, 스캔 이미지, 수기 양식 등 형태가 다양합니다. Apple Vision 모델로 1차 텍스트를 추출하고, Gemini 멀티모달 모델로 이미지와 텍스트를 동시에 해석하는 2단계 파이프라인을 구성했습니다.
- **비IT 사용자 UX**: Obsidian의 Markdown 기반 인터페이스를 활용해 최소한의 학습으로 접근할 수 있도록 설계했습니다. 자동화 스크립트가 백그라운드에서 동작하여 사용자는 기존 워크플로우를 크게 바꾸지 않아도 되었습니다.
- **다현장 데이터 통합**: 현장마다 다른 분류 체계와 메타데이터를 표준화된 Markdown 구조로 통합하여, 전 현장의 현황을 한 곳에서 조회할 수 있게 했습니다.

## Outcome

- **약 5개 현장 통합 문서 중앙 관리** 체계를 구축했습니다.
- **OCR + LLM 기반 문서 자동 등록 및 요약** 흐름을 구현하여 비정형 문서를 구조화했습니다.
- **월간보고서 자동 생성 파이프라인**을 도입해 반복 업무 부담을 줄였습니다.
- 사업비·계약 데이터의 **실시간 대시보드**를 통해 의사결정 속도를 개선했습니다.
- 결과적으로 **전 현장의 핵심 협업 시스템으로 정착**시켜, 기술 도입과 현장 안착을 동시에 검증했습니다.

### 이 프로젝트가 보여주는 것

이 프로젝트는 건설 현장의 실제 문제를 AI 협업 시스템으로 번역할 수 있는 능력을 검증한 사례입니다. 도메인 맥락 해석, 이해관계자 요구 분석, 비IT 사용자 대상 시스템 설계, 그리고 AI 파이프라인 구축까지 — 건설CM 경험과 AI 기술을 연결하는 bridge 역할을 수행했습니다.
