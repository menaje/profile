01

문제의식

현장에서 반복된 손실을 기술의 질문으로 바꿨습니다

2015.07 ~ 2024.05 8년 10개월 CM 47건 구조 기술지원

기술로 방향을 틀게 된 출발점은 새로운 유행이 아니라 현장의 반복 손실이었습니다. 신탁사업 설계단계 구조 검토를 47건 지원하던 시기, 비슷한 유형의 검토가 다시 들어와도 이전 회의 메모와 판단 근거가 담당자 개인의 폴더와 기억에 흩어져 있어 같은 설명을 처음부터 다시 꺼내야 하는 장면을 여러 번 겪었습니다.

저는 이 문제가 단순한 비효율이 아니라 지식 노동 전체를 느리게 만드는 구조라고 생각하게 됐습니다. 기술은 화려한 기능의 이름이 아니라, 흩어진 정보와 암묵지를 다시 연결해 실제 업무 흐름을 바꾸는 방법이었습니다.

02

커리어 전환

문제가 기술을 이끌었고, 자동화가 전환의 기준이 됐습니다

2017 ~ 2024 13개 VE 프로젝트 자동화 3건 협업 도구 도입 5개 현장 문서관리 체계

전환은 갑작스러운 선언이 아니라 현장의 작은 자동화의 연속이었습니다. VE 업무를 정리하며 13개 프로젝트에 보고서 자동 생성 기반을 적용했고, Trello와 REDMINE을 직접 제안·운영하며 도구를 들이는 것과 업무 흐름이 바뀌는 것의 차이를 배웠습니다.

마지막으로 5개 현장을 아우르는 문서관리 체계를 설계하면서 문서 등록, 요약, 월간보고를 하나의 흐름으로 묶을 수 있다는 점을 확인했습니다. 비IT 배경에서 개발을 익히는 과정은 쉽지 않았지만, 어떤 정보를 먼저 구조화해야 하는지 집요하게 따지는 태도가 전환의 핵심 동력이었습니다.

03

도전 사례

제안서 에이전트로 여러 산출물을 하나의 흐름으로 묶었습니다

2024.05 ~ 2025.11 20여 종 산출물 흐름 설계 26개 API · 16개 서비스 · 15개 DB 테이블

이 과제의 본질은 문서를 빠르게 만드는 데 있지 않았습니다. 한 프로젝트 안에서 여러 보고서와 슬라이드가 동시에 만들어질 때, 앞쪽 요약에서 약속한 내용과 뒤쪽 기술 설명이 어긋나지 않도록 전체 산출물이 하나의 흐름으로 읽히게 만드는 일이 더 중요했습니다.

저는 작업을 Phase→Stage→Task 세 계층으로 나누고, 각 단계의 결과를 다음 단계의 입력으로 누적 전달하는 방식을 설계했습니다. 그 결과 26개 API와 16개 서비스, 15개 DB 테이블로 구성된 체계 안에서 20여 종의 산출물이 연결된 흐름으로 생성되는 구조를 만들 수 있었습니다.

04

도전 사례

CONI로 끊기지 않는 협업의 연속성을 설계했습니다

2025.11 ~ 진행 중 (2026-03-20 기준) 개인 프로젝트 5단계 파이프라인 · 6가지 비정상 상태 대응

제안서 에이전트를 만들며 또 다른 질문이 생겼습니다. 왜 AI와의 협업은 대화가 끊기는 순간 처음부터 다시 시작해야 하는가. 어제의 판단과 오늘의 실행이 세션이 바뀌는 순간 사라진다면, 아무리 좋은 모델이 있어도 팀의 생산성을 안정적으로 높이기 어렵다고 느꼈습니다.

CONI에서는 작업 계획, 진행 상태, 결과 이력을 Objective→Workplan→Commit 구조로 남기고, 5단계 작업 파이프라인을 따라 다시 이어질 수 있게 만들었습니다. 이후에는 6가지 비정상 상태를 유형화하고 각각에 대해 다음 행동을 제안하는 fail-safe 정책을 설계했습니다. 이 프로젝트는 협업의 연속성을 시스템으로 바꿔 본 경험이라는 점에서 특히 중요했습니다.

미래 비전

도메인 지식과 데이터의 복잡성을 실전형 AI 시스템으로 풀고 싶습니다

건축사업 데이터 플랫폼 제안서 8종 설계→CM→입찰→실행→운영 판단 근거의 연속성

앞으로도 저는 도메인 지식이 깊고 데이터가 복잡하게 얽힌 문제를 AI 시스템으로 풀고 싶습니다. 설계에서 CM으로, 입찰에서 실행으로, 실행에서 운영으로 넘어갈 때마다 문서만 전달되는 것이 아니라 판단의 근거와 변경의 배경까지 함께 이어지는 시스템을 만들고 싶습니다.

제가 만들고 싶은 것은 기술을 과시하는 도구가 아니라, 데이터가 스스로 일할 수 있도록 돕는 실전형 시스템입니다. 현장을 이해하는 사람의 문제의식과 개발자의 구현 능력이 한 방향을 볼 때, AI는 보여주기용 기능이 아니라 조직의 실제 생산성을 바꾸는 기반이 될 수 있다고 믿습니다.
