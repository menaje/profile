Pilot Review

01

해안건축사사무소 건축사업 데이터 연결

같은 프로젝트는 같은 흐름으로 이어져야 합니다

부서별 업무 방식은 유지하되, 같은 프로젝트의 정보와 판단 배경은 하나의 흐름으로 이어지게 합니다.

업무 방식은 그대로 판단 배경은 이어서 파일럿으로 검토
파일럿 검토안

Problem

02

프로젝트는 하나인데 설명은 여러 번 반복됩니다

단계가 바뀔 때마다 같은 프로젝트를 다시 설명하고, 다시 정리하고, 다시 확인하게 됩니다.

현장 부담

  • 단계가 바뀔 때마다 같은 회의가 다시 시작됩니다.
  • 이미 본 쟁점을 처음부터 다시 풀어 말합니다.
  • 한 프로젝트에서 3~4회 이상 같은 확인이 반복됩니다.

조직 부담

  • 전환 구간마다 1~2주의 재파악 기간이 생깁니다.
  • 부서 간 협업의 리듬이 전환 시점에서 자주 끊깁니다.
  • 판단 속도가 불필요하게 늦어져 일정이 흔들립니다.
같은 내용을 다시 푸는 시간이 누적됩니다

Structure

03

문서는 남아도 판단의 배경은 함께 넘어가지 않습니다

자료는 남습니다

  • 문서는 전달됩니다.
  • 변경 흔적도 일부 남습니다.
  • 결론만 보면 정리된 듯 보입니다.

그러나 흐름은 끊깁니다

  • 왜 이렇게 정리됐는지
  • 무엇이 아직 남아 있는지
  • 어디서부터 봐야 하는지

문제의 핵심은 자료 부족이 아니라, 맥락이 함께 넘어가지 않는 구조입니다.

전달은 되지만 이어받음은 약합니다

Scene

04

현장에서는 같은 쟁점을 다시 설명하게 됩니다

설계 검토가 끝난 뒤 다음 단계 담당자가 들어오면, 자료는 있어도 왜 이 방향으로 정리됐는지부터 다시 묻게 됩니다.
다시 설명 다시 확인 다시 메모
자료 부족이 아니라 맥락 단절의 문제입니다

Proposal

05

제안의 본질은 세 문장으로 정리됩니다

1. 그대로 두는 것

각 부서는 기존 방식대로 일합니다.

2. 이어지게 하는 것

프로젝트 데이터의 맥락은 끊기지 않게 이어집니다.

3. 시작하는 방식

처음부터 크게가 아니라 파일럿으로 작게 시작합니다.

세 문장이 이번 제안의 중심축입니다

Keep

06

각 부서는 기존 방식대로 일합니다

바꾸지 않는 것

  • 설계와 CM, 입찰과 실행의 관점 차이
  • 부서별 역할과 책임
  • 현장의 기존 판단 기준

맞추는 것은 하나입니다

  • 넘겨받을 때 필요한 기준
  • 이전 판단의 배경
  • 미결 사항의 위치
통합된 방식 강요가 아니라 연결 기준의 정리입니다

Handoff

07

다음 단계는 이미 쌓인 흐름 위에서 시작합니다

이전 방식

  • 문서를 다시 읽고
  • 빠진 배경을 다시 묻고
  • 핵심 쟁점을 다시 찾습니다.

이어받는 방식

  • 이미 쌓인 판단 위에서 시작하고
  • 미결 사항을 바로 확인하고
  • 핵심 쟁점을 더 빨리 봅니다.
전달이 아니라 이어받음이 핵심입니다

Meaning

08

역할이 많이 이어질수록 연결의 차이가 커집니다

@@주요사업영역@@ @@조직특성@@

이런 특성을 가진 조직일수록 설명 부담이 줄고, 판단의 흐름이 이어지는 차이가 더 크게 드러납니다.

의미 1

부서장이 핵심 쟁점을 더 빨리 봅니다.

의미 2

실무자가 배경을 다시 복원하는 시간이 줄어듭니다.

의미 3

프로젝트를 하나의 흐름으로 보는 힘이 커집니다.

세부 확인 전 정보는 슬롯 그대로 유지합니다

Pilot

09

시작은 크게가 아니라 작게입니다

반복 설명이 가장 많이 생기는 전환 구간 하나를 정해, 실제로 이어받기가 나아지는지 먼저 확인합니다.

범위

한 구간만 먼저 봅니다.

목표

실제로 나아졌는지 확인합니다.

판단

효과가 보이면 그다음 확장을 논의합니다.

전사 확대보다 먼저 파일럿 검토를 합니다

Start Point

10

첫 파일럿은 전환 구간 하나면 충분합니다

기본안

설계 → CM

가장 먼저 검토할 기본 축입니다.

조정 가능 구간

@@인수인계구간@@

실제로 더 시급한 지점이 있다면 한 구간만 먼저 검토합니다.

@@인수인계구간@@은 확인 전까지 그대로 둡니다

Measure

11

효과는 체감되는 변화로 판단합니다

반복 설명

같은 내용을 다시 풀어 말하는 횟수가 절반 이하로 줄어드는가

쟁점 파악

핵심 쟁점을 파악하는 데 걸리는 시간이 기존 대비 반 이상 단축되는가

미결 사항

남아 있는 판단거리가 목록으로 바로 확인되어 누락이 줄어드는가

배경 복원

다음 단계 담당자의 재파악 기간이 1~2주에서 2~3일 이내로 줄어드는가

과한 약속보다 현장에서 느끼는 개선을 봅니다

Action

12

오늘 회의에서 정하면 되는 것은 세 가지입니다

  1. 우선 대상 구간

    어느 전환 구간을 먼저 볼지

  2. 참여 부서

    누가 함께 검토할지

  3. 확인할 기준

    무엇이 나아지면 의미가 있다고 볼지

부서별 업무 방식은 유지하고, 같은 프로젝트는 같은 흐름으로 이어지게 하자는 제안입니다.

세 가지가 정해지면 파일럿 착수 여부를 바로 논의할 수 있습니다
