업무 유형별 AI 전략

설계와 CM은 공통적으로
데이터 기반이 먼저 필요합니다.

다만 그 위에서 AI에 기대하는 기능은 다릅니다. 설계는 초기 기획·검토와 실제 수행 단계에서 요구가 나뉘고, CM은 데이터 연결과 근거 재구성의 비중이 더 큽니다.

단계별 혼합형 architecture

설계

설계는 데이터 기반이 전제되며, 초기 기획·검토 단계에서는 데이터 연결의 영향이 크고 실제 설계 수행 단계에서는 도구와 생성 성능의 영향이 더 크게 나타납니다.

업무 성격

초기 기획·검토와 실제 설계 수행이 이어지는 흐름

주요 산출물

설계도서, 대안안, 검토 결과물

AI 관점

기획·검토는 데이터 연결이 중요하고, 수행은 높은 생성 성능과 도구 활용성이 중요

핵심 포인트

데이터 기반은 공통 전제이고, 설계는 단계에 따라 데이터 연결과 도구·생성 성능의 비중이 달라집니다.

데이터 연결·근거 재구성형 description

CM

CM 역시 데이터 기반이 전제되어야 하며, 공문·기술검토서 같은 작은 문서를 반복 생산하는 업무 특성상 데이터 연결과 근거 재구성 체계의 중요도가 더 높습니다.

업무 성격

사안별 작은 문서를 반복 생산하는 흐름

주요 산출물

공문, 기술검토서, 회의록, 변경·승인 기록

AI 관점

중급 모델이어도 데이터 관리 체계와 활용 전략이 더 중요

핵심 포인트

데이터 기반은 공통 전제이고, 문서·이슈·승인 데이터의 연결성이 문서화 품질을 좌우

balance

설계와 CM 모두 데이터 기반이 먼저 필요합니다. 다만 설계는 초기 기획·검토와 실제 수행 단계에 따라 데이터 연결과 도구·생성 성능의 비중이 달라지고, CM은 데이터 관리 체계와 근거 기반 문서화를 더 우선적으로 설계해야 합니다.
